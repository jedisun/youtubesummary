# Unicode 时序图输出规范

## 1. 目标

本规范用于统一本仓库中的 Unicode 文本时序图写法。

适用场景：

- 设计文档中的处理流程说明
- 异步/同步调用关系说明
- 任务队列、状态推进、结果回写说明

目标：

1. 让调用方向清晰
2. 让返回时机清晰
3. 让同步/异步语义清晰
4. 让图可以直接映射到代码实现

---

## 2. 基本格式

时序图使用纯文本 Unicode 字符绘制，不使用 Mermaid。

推荐基础结构：

```text
NodeA           NodeB           NodeC
  │               │               │
  │ call_x()      │               │
  │──────────────>│               │
  │<──────────────│               │
  │ result_x      │               │
  │               │ call_y()      │
  │               │──────────────>│
  │               │<──────────────│
  │               │ result_y      │
```

---

## 3. 对齐规则

### 3.1 节点对齐

- 每个节点名称独占一列
- 节点名称下方的生命线 `│` 必须与该节点中心对齐
- 同一列中的所有 `│` 必须垂直对齐

### 3.2 箭头对齐

- 调用箭头的起点和终点必须明确落在对应节点的生命线上
- 不允许出现“看不出箭头到底指向哪个节点”的情况
- 如果节点太多，必须拆成多张小时序图，不要硬塞成一张超宽图

### 3.3 返回值对齐

- 返回箭头单独占一行
- 返回值放在**返回箭头的正下方**
- 不要把返回值挂在图的最右侧
- 不要把返回值写在调用箭头旁边

---

## 4. 调用与返回规则

### 4.1 调用文本位置

- 函数调用名写在**调用箭头上方**
- 一条箭头只表达一次调用
- 不要在一条箭头上写多个动作

示例：

```text
  │ create_task()  │
  │──────────────> │
```

### 4.2 返回文本位置

- 返回结果写在**返回箭头的下方**
- 返回内容要体现真实语义，例如：
  - `stream_url`
  - `accepted`
  - `segment_list`
  - `summary_block`
  - `written`
  - `ok`

示例：

```text
  │<────────────── │
  │ result_value   │
```

---

## 5. 同步与异步表达规则

同步和异步的区别**不要通过不同箭头样式表达**，而要通过“返回值语义”表达。

### 5.1 同步调用

同步调用返回的是“真实处理结果”。

示例：

```text
Watcher         Transcriber
  │ transcribe_file(chunk_path)
  │──────────────>
  │<──────────────
  │ segment_list
```

含义：

- 调用发出后，需要等待处理完成
- 返回值就是实际结果

### 5.2 异步调用

异步调用只返回“已接收 / 已入队”，真正处理在后续时序中发生。

示例：

```text
Watcher         Transcriber
  │ enqueue_transcribe(chunk_path)
  │──────────────>
  │<──────────────
  │ accepted

...稍后...

Transcriber     ChunkDir
  │ read_chunk(chunk_path)
  │──────────────>
```

含义：

- `accepted` 不代表处理完成
- 只是代表任务已被接收
- 后续处理必须单独画出

---

## 6. 拆图规则

满足任一条件时，必须拆成多张小时序图：

1. 节点过多，箭头终点不清楚
2. 一张图里同时包含过多异步分支
3. 阅读时无法一眼区分“输入层”和“处理层”
4. 调用和返回开始互相遮挡

推荐拆分方式：

- 图 A：输入 -> 发现 -> 转写
- 图 B：缓冲 -> 聚合 -> 摘要 -> 落盘

---

## 7. 命名规则

- 节点名使用简短英文或中英混合短名
- 调用名优先使用拟实现的函数名
- 返回值使用结果语义，不使用笼统描述

推荐：

- `resolve_stream()`
- `write_chunk()`
- `scan_chunks()`
- `is_chunk_stable()`
- `enqueue_transcribe()`
- `transcribe_file()`
- `build_window()`
- `enqueue_summary()`
- `summarize_window()`
- `append_summary_block()`
- `update_window_state()`

不推荐：

- `处理数据`
- `开始任务`
- `执行流程`
- `返回结果`

---

## 8. 输出规则

以后在本仓库输出时序图时，默认遵守以下规则：

1. 优先使用 Unicode 文本图
2. 节点与生命线严格对齐
3. 调用名写在调用箭头上方
4. 返回值写在返回箭头下方
5. 同步/异步区别通过返回语义表达
6. 异步任务的后续处理必须单独画出
7. 图太宽就拆分，不强行压缩成一张

---

## 9. 推荐模板

```text
NodeA           NodeB           NodeC
  │               │               │
  │ call_a()      │               │
  │──────────────>│               │
  │<──────────────│               │
  │ result_a      │               │
  │               │ enqueue_b()   │
  │               │──────────────>│
  │               │<──────────────│
  │               │ accepted      │

...稍后...

  │               │               │ do_b()
  │               │<──────────────│
  │               │ result_b      │
```
